# DeepGuard Observatory - 한국어

공개 플랫폼에서 딥페이크 관련 콘텐츠를 탐지하고 추적하는 커뮤니티 중심 모니터링 시스템입니다.

## 개요

DeepGuard Observatory는 Reddit, GitHub와 같은 공개 플랫폼을 모니터링하여 비합의 합성 미디어(딥페이크) 생성에 악용될 수 있는 AI 모델과 기술을 탐지합니다. 유해한 콘텐츠가 발견되면 공식 채널을 통해 신고하고, 투명한 통계를 유지합니다.

이것은 한 명이 지속적으로 운영할 수 있도록 설계된 주말 프로젝트입니다. 목표는 겸손하지만 명확합니다. 우리가 볼 수 있는 딥페이크를 잡아내고 막으며, 발견한 내용을 기록하고, 데이터가 말하도록 두는 것입니다.

## 문제 정의

딥페이크 기술은 실제 피해를 입힙니다. 특히 여성과 아동이 피해자가 되는 경우가 많습니다. 대형 AI 기업들은 내부 안전팀을 갖추고 있지만, 오픈소스 커뮤니티는 자신의 도구가 어떻게 악용되고 있는지에 대한 조율된 가시성이 부족합니다. 어떤 플랫폼이 가장 빠르게 대응하는지, 어떤 패턴이 나타나는지, 생태계가 실제로 어떻게 진화하고 있는지에 대한 공개 기록이 없습니다.

이 프로젝트는 간단하고 정직한 모니터링으로 그 공백을 채웁니다.

## 작동 방식

### 흐름

1. 스캔: Reddit과 GitHub에서 딥페이크 관련 콘텐츠 일일 자동 확인
2. 분석: Claude API를 사용하여 콘텐츠가 실제로 유해한지 판단
3. 신고: 공식 플랫폼 채널을 통해 제출 (Reddit 신고, GitHub 이슈)
4. 추적: 발생한 일을 기록하고 월간 통계 발행
5. 반복: 설정 후 하루에 한 번 스캔, 인간 개입 불필요

### 신고 대상

- 합성 미디어 생성을 명시적으로 목표로 하는 GitHub 저장소
- 안전 필터 우회 기술 및 코드
- 유해한 모델 또는 데이터셋 모음
- 비합의 합성 방법에 대한 단계별 가이드

### 신고하지 않는 항목

- 합법적인 학술 딥페이크 탐지 연구
- 라이센스된 엔터테인먼트 사용 사례
- 이론적 보안 논의
- 적절한 면책사항과 안전 조치가 있는 저장소

## 시작하기

### 사전 요구사항

- Python 3.9+
- Reddit API 자격증 (무료, 5분 소요)
- Anthropic API 키 (유료, 기본 비용 절감 설정 기준 월 1-5달러 — 아래 참고)

### 설정 (30분)

```bash
# Clone
git clone https://github.com/[your-username]/deepguard-observatory.git
cd deepguard-observatory

# 환경
python -m venv venv
source venv/bin/activate  # macOS/Linux
# 또는: venv\Scripts\activate  # Windows

# 의존성
pip install -r requirements.txt

# 설정
cp .env.example .env
# .env에서 API 키 입력
```

### API 키 받기

**Reddit API** (무료):
1. https://www.reddit.com/prefs/apps 이동
2. "Create an application" 클릭
3. "script" 타입 선택
4. Client ID와 Client Secret 복사

**Claude API** (유료):
1. https://console.anthropic.com/ 이동
2. API 키 생성
3. .env에 추가

### 실행

```bash
# 한 번만 스캔
python src/main.py

# 또는 일일 자동화 설정 (GitHub Actions)
# .github/workflows/daily_scan.yml의 설정 따르기
```

## API 비용 절감 설계

1인 프로젝트이므로 파이프라인이 최대한 적게 쓰도록 설계되어 있습니다:

1. **중복 제거** — 이미 분석한 항목의 ID를 `data/seen_ids.json`에 저장해서,
   매일 실행해도 *새로운* 콘텐츠에만 비용을 지불합니다. 이게 가장 큰 절감
   포인트입니다. 없으면 매번 같은 최근 게시물을 다시 분석하게 됩니다.
2. **2단계 분석** — 모든 후보를 저렴한 모델(Haiku 4.5, 100만 토큰당 $1/$5)로
   먼저 선별하고, 유해 판정이 나오거나 확신이 낮은 항목만 강한 모델
   (Opus 4.8, $5/$25)로 재확인합니다. 후보 대부분은 노이즈라 비싼 모델은
   드물게만 돌아갑니다. `.env`의 `SCREEN_MODEL` / `CONFIRM_MODEL`로 조정.
3. **입력 길이 제한** — 게시물 본문은 1,500자로 잘라서 보냅니다.
4. **사용량 로깅** — 매 실행마다 토큰 사용량과 예상 비용이 출력되어
   비용이 늘어나면 바로 보입니다.

추가 권장: [Anthropic Console](https://console.anthropic.com/)에서 월 지출
한도를 설정해 두세요 (최후의 안전장치).

하루 신규 항목이 10-30개 수준이면 **월 1-5달러** 정도입니다. 규모가 10배
커지면 다음 수단은 [Message Batches API](https://platform.claude.com/docs/en/build-with-claude/batch-processing)
(50% 할인, 1시간 내 결과 — 일일 크론에 딱 맞음)입니다.

## 프로젝트 구조

```
deepguard-observatory/
├── src/
│   ├── main.py              # 진입점
│   ├── reddit_scanner.py    # Reddit 키워드 스캔
│   ├── github_scanner.py    # GitHub 저장소 검색
│   ├── analyzer.py          # 2단계 Claude 분석
│   └── state.py             # 중복 제거 상태 (seen IDs)
├── reports/                 # 실행별 스캔 결과 (scan-*.json)
├── data/                    # 중복 제거 상태 (seen_ids.json)
├── .github/workflows/
│   └── daily_scan.yml       # GitHub Actions 자동화
├── requirements.txt
├── .env.example
├── README.md
└── LICENSE
```

## 사용법

### 수동 스캔

```bash
python src/main.py
```

출력:
- 결과가 포함된 콘솔 로그 (토큰 사용량/예상 비용 포함)
- reports/scan-[타임스탬프].json 생성
- data/seen_ids.json 업데이트

### 자동화된 일일 스캔

1. GitHub Secrets에 API 키 설정 (`REDDIT_CLIENT_ID`, `REDDIT_CLIENT_SECRET`, `ANTHROPIC_API_KEY`)
2. 저장소 설정에서 GitHub Actions 활성화
3. 워크플로우 매일 UTC 오전 9시 실행
4. 결과 자동 커밋 (reports/, data/)

## 데이터 및 투명성

모든 발견 사항은 공개 저장소에 다음과 함께 기록됩니다:
- 스캔된 플랫폼
- 발견된 유해 항목 수
- 성공적으로 신고된 항목 수
- 플랫폼이 대응하는 데 걸린 시간
- 플랫폼별 응답률

개인 정보는 저장되지 않습니다. 분석은 콘텐츠에만 초점을 맞춥니다.

## 성공 지표

### 직접 영향 (월간)

- 탐지 항목: 목표 15-30개
- 신고 항목: 목표 80% 이상 성공
- 플랫폼 제거율: 목표 60% 이상

### 생태계 영향

- GitHub 별: 첫 3개월 20-50개
- 월간 리포트 읽기: 100-500명
- 커뮤니티 참여: 2-5개 응답

### 개인적 성과

- 실제 API 통합 능력 입증
- 데이터 기반 의사결정 방식 시연
- AI 안전에 대한 헌신 증명
- 반복 가능하고 감시 가능한 시스템 구축

## 법적 및 윤리적 가이드라인

### 우리가 하는 것

✓ 공개적으로 이용 가능한 정보만 모니터링
✓ 공식 플랫폼 신고 채널 사용
✓ 익명이고 사실에 기반한 신고 유지
✓ 방법론을 투명하게 공개
✓ 수정 및 피드백 허용

### 우리가 하지 않는 것

✗ 개인 정보 수집 또는 노출
✗ 불법 콘텐츠 다운로드 또는 저장
✗ 자동화된 스팸 또는 괴롭힘
✗ 증거 없이 주장하기
✗ 특정 개인 또는 회사 표적화

- 모든 활동이 기록되고 감시 가능

- 법적 검토에 열려 있음

## 한계

이 프로젝트는:
- 모든 딥페이크를 예방할 수 없습니다 (공개적으로 보이는 것만 잡을 수 있음)
- 플랫폼의 협력에 따라 제거 여부가 결정됩니다
- API가 있는 플랫폼으로만 제한됩니다
- 효과적이려면 지속적인 모니터링이 필요합니다

이것은 괜찮습니다. 목표는 완벽하지 않고 유용한 것입니다.

## 왜 이것이 중요한가?

예방된 모든 딥페이크는 실제 피해로부터 보호받는 사람입니다. 이 프로젝트가 그렇지 않으면 생성되었을 딥페이크의 1%를 잡아내고 멈춘다면, 노력할 가치가 있습니다.

부수적 목표는 분산된 개인 노력이 변화를 만들 수 있다는 것을 증명하는 것입니다. 안전성에서 진전을 이루기 위해 기업이 필요하지 않습니다.
